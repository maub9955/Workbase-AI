# Render Web 서비스 접속 문제 해결

## 문제
- `workbase-ai.com`에 접속이 안 됨
- Web 앱이 작동하지 않음

## 즉시 확인할 사항

### 1. Render Dashboard에서 Web 서비스 상태 확인

1. Render Dashboard 접속
2. Web 서비스 선택 (API 서버가 아닌 Web 앱 서비스)
3. 서비스 상태 확인:
   - ✅ "Live" 또는 "Running"인지 확인
   - ❌ "Building" 또는 "Failed"인지 확인

### 2. 배포 로그 확인

1. **Logs** 탭 클릭
2. 최신 배포 로그 확인:
   - 빌드가 성공했는지
   - 서버가 시작되었는지
   - 오류 메시지가 있는지

### 3. 커스텀 도메인 확인

1. **Settings** → **Custom Domains** 섹션
2. `www.workbase-ai.com` 도메인이 연결되어 있는지 확인
3. DNS 설정이 올바른지 확인

### 4. 서비스 재시작

서비스가 일시 중지되었을 수 있습니다:

1. **Manual Deploy** → **Deploy latest commit** 클릭
2. 배포 완료까지 대기 (보통 2-3분)
3. 서비스가 "Live" 상태가 되는지 확인

## 일반적인 원인

### 1. 서비스 일시 중지
- Render 무료 플랜은 15분 비활성 후 일시 중지
- 첫 요청 시 자동 재시작 (약간의 지연)

### 2. 빌드 실패
- 환경 변수 문제
- 빌드 오류
- 의존성 문제

### 3. DNS 문제
- 커스텀 도메인 설정 오류
- DNS 전파 지연

### 4. 서비스 크래시
- 런타임 오류
- 메모리 부족

## 해결 방법

### 방법 1: 서비스 재배포

1. **Manual Deploy** → **Deploy latest commit**
2. 배포 완료까지 대기
3. 서비스 상태 확인

### 방법 2: 로그 확인

1. **Logs** 탭에서 오류 메시지 확인
2. 오류가 있다면 공유하여 해결

### 방법 3: 환경 변수 확인

1. **Environment** 섹션에서 환경 변수 확인
2. `NEXT_PUBLIC_API_URL`이 올바르게 설정되어 있는지 확인

## 다음 단계

1. Render Dashboard에서 Web 서비스 상태 확인
2. 배포 로그 확인
3. 필요시 서비스 재배포
4. 결과 공유
